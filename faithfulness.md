# Faithfulness Harness

The faithfulness harness compares the local kernel fragment against observed Lean 4 behavior on small source examples.  It does not make Lean 4 the project specification, because the local specification remains the authority for this repository.  It records agreement and disagreement with Lean 4 as evidence, and every resulting decision still needs a written rule in `spec.md`.

The first corpus lives under `Faithfulness`.  `Accepted.lean` contains Lean source that the installed Lean compiler must accept, including universe-polymorphic definitions, theorem declarations, inductive recursors, proposition-valued recursors, equality elimination, low-level quotients, mutual inductives, nested mutual inductives, indexed structure projections, inherited structures, transparent definitions, abbreviations, and opaque constants.  The files under `Faithfulness/Rejected` contain Lean source that the installed Lean compiler must reject, including a non-positive inductive, a theorem whose type is not a proposition, an ambiguous `Sort u` inductive result, forbidden eliminations from propositions into data, invalid projections, failed eta across computed indices, an opacity-sensitive `rfl`, a quotient relation mismatch, and a nested inductive parameter that contains a local variable.

The executable `lake exe leanleanfaith` runs the corpus through the installed `lean` command.  The accepted file must exit successfully, and each rejected file must fail with an expected diagnostic fragment.  The environment variable `LEANLEAN_LEAN` can name a specific Lean binary when the default command does not select the intended compiler.

The local bridge lives in the ordinary regression suite and in `lake exe leanleanimport`.  The regression suite checks corresponding local expressions for the same behavioral classes, including raw literal translation and natural-literal computation.  The importer smoke test loads selected roots from the accepted corpus through Lean's environment, extracts their `ConstantInfo` closures, and replays them through the local declaration importer.  It includes ordinary definitions, opaque definitions, propositions, mutual inductives, nested mutual inductives, structures and projections, equality, quotients, core inductives, and indexed local inductives.
