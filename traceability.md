# Traceability Map

This map connects the implemented specification to the code paths and tests that exercise each trusted rule.  It records evidence, not a metatheoretic proof.  A row is complete only when the specification clause has an implementation entry point and at least one regression or Lean comparison path.

## Coverage Gaps

The map still lacks theorem-level statements for preservation, progress, conversion soundness, positivity soundness, and recursor correctness.  The implementation has executable checks for those rules, but the repository does not yet contain a mathematical proof that the rules imply consistency or that replay preserves Lean 4 kernel acceptance on the named fragment.  The next artifact should therefore state the metatheoretic claims in enough detail that each row above can be audited against an explicit lemma or theorem.

The Lean comparison evidence remains finite.  `leanleandiff` now checks named environment fragments, but it does not quantify over every declaration in Lean's prelude or every module reachable from an arbitrary user file.  Extending that boundary requires new specification clauses for the divergence table before adding broader replay roots.
